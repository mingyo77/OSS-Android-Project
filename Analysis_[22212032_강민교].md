## 1. Introduction

본 문서는 주방 운영의 디지털 전환을 목표로 하는 'Helper' 시스템의 상세 분석 내용을 담고 있습니다. 'Helper'는 고온 및 소음이 발생하는 조리 환경에서 발생할 수 있는 인적 오류(Human Error)를 최소화하고, 신입 인력의 빠른 업무 적응을 돕기 위해 설계되었습니다. 본 프로젝트의 주요 특징은 다음과 같습니다.

* **유용성 (Usefulness):** 초 단위의 정밀 조리 타이머와 직관적인 레시피 가이드를 제공하여, 작업자의 숙련도와 관계없이 항상 일관된 음식 품질을 보장합니다.
* **중요성 (Significance):** 기존의 구두 교육과 개인의 감에 의존하던 전통적 주방 문화를 시스템 기반의 데이터 관리 체계로 전환하여, 운영 리스크를 줄이고 매장 관리의 효율성을 극대화합니다.
* **확장성 (Expandability):** 향후 재고 관리 시스템 및 본사 매뉴얼 업데이트 기능 등과 연동하여, 개별 매장을 넘어선 통합 프랜차이즈 관리 솔루션으로 확장할 수 있는 잠재력을 가집니다.

Conceptualization 보고서에 작성한 유즈케이스를 바탕으로 starUML을 이용해서 각 UseCase들을 서로 어떤 관계인지 구분하여 연결지었습니다.

### 2) Use case description

#### [UC-01] Login
* **Summary:** 사용자가 시스템을 이용하기 위해 자신의 역할(알바생/관리자)에 맞는 계정으로 인증을 수행한다.
* **Scope:** Helper Application
* **Level:** User goal
* **Primary Actor:** Part-timer, Manager
* **Preconditions:** 안드로이드 기기에 앱이 설치되어 있고, DB 연결이 가능한 상태여야 한다.

#### [UC-02] Login Error
* **Summary:** 잘못된 계정 입력 시 다이얼로그 예외 처리를 수행한다.

#### [UC-03] View Notice
* **Summary:** 홈 화면에서 오늘의 주요 공지사항을 조회한다.

#### [UC-04] View Unfinished Task
* **Summary:** 홈 화면에서 남은 업무 개수를 시각적으로 확인한다.

#### [UC-05] Manage Notice
* **Summary:** 관리자가 매장 공지를 작성/수정/삭제한다.

#### [UC-06] Todo Task
* **Summary:** 체크리스트 항목을 완료 상태로 변경한다.

#### [UC-07] Manage Task Template
* **Summary:** 반복 업무(오픈/마감 등) 템플릿을 추가/편집/삭제한다.

#### [UC-08] Search Recipe
* **Summary:** 조리 스펙(시간, 온도 등)을 검색 및 세팅한다.

#### [UC-09] Execute Timer
* **Summary:** 초 단위 정밀 타이머를 구동하고 알람을 발생시킨다.

#### [UC-10] Edit Recipe Data
* **Summary:** 관리자가 메뉴 조리 시간/온도를 수정한다.

#### [UC-11] Manage Staff
* **Summary:** 알바생 계정을 등록(사번 발급) 및 삭제한다.

#### [UC-12] View Profile
* **Summary:** 내 정보 확인 및 로그아웃을 수행한다.

## 3. Domain analysis

### 1) User
시스템을 이용하는 사용자의 계정 정보와 권한(매니저/알바생)을 관리하는 핵심 기능들이 담긴 클래스이다.

### 2) Notice
매니저가 등록한 매장 내 공지사항의 내용을 저장하고 불러오는 데 필요한 핵심 기능들이 담긴 클래스이다.

### 3) TaskTemplate
반복되는 매장의 루틴 업무(오픈/마감 등)의 기본 틀을 정의하고 관리하는 핵심 기능들이 담긴 클래스이다.

### 4) Task
할 당일 업무들을 확인하고 수행 상태를 관리하는 과정에서 필요한 핵심 기능들이 담긴 클래스이다.
* **ex)** 업무 완료로 체크하기, 체크 해제하기, 오늘 남은 업무 개수 계산하기...

### 5) Recipe
알바생들이 조리 전 메뉴별 매뉴얼을 검색하고 스펙을 확인하는 과정에서 필요한 핵심 기능들이 담긴 클래스이다.
* **ex)** 레시피 이름으로 검색하기, 메뉴별 기본 조리시간/온도 불러오기, 레시피 내용 수정하기...

### 6) CookingTimer
알바생들이 레시피에 맞춰 정확한 조리 시간을 측정하고 관리하는 과정에서 필요한 핵심 기능들이 담긴 클래스이다.
* **ex)** 타이머 시작하기, 타이머 정지하기, 남은 시간 계산해서 화면에 보여주기...

### 7) Dashboard
사용자들이 홈 화면(메인 화면)에서 당일 근무 진행 상황을 한눈에 파악하는 과정에서 필요한 핵심 기능들이 담긴 클래스이다.
* **ex)** 전체 업무 달성률 계산하기, 미완료 업무 요약해서 보여주기...

## 4. User Interface prototype

### 1) Login Screen
앱을 실행하면 가장 먼저 나오는 로그인 화면입니다.
매니저에게 부여받은 번호로 입장 가능하며, 클릭하면 홈으로 들어갑니다.

### 2) Home Screen
홈 화면에 대한 프로토타입으로 부여받은 사원 번호로 로그인하면 해당 사원의 이름이 뜨며 해야할 일과 매니저가 작성한 공지사항을 확인하게 됩니다.

### 3) Task Screen
업무 화면의 프로토타입으로 완료 상태와 해야 할 일의 현재 완료 여부가 노출되어 있습니다. 

### 4) Cooking Screen
요리 화면의 프로토타입입니다.

## 5. Glossary

### [1] 도메인 및 비즈니스 용어 (Domain & Business)
* **레스팅 (Resting):** 튀김 조리 후 고기 내부의 육즙이 골고루 퍼지도록 잠시 대기하는 필수 과정.
* **표준화 (Standardization):** 작업자 개인의 편차와 상관없이 동일 품질의 결과물을 도출하기 위한 공정 규격화.

### [2] 기술 및 시스템 용어 (Tech & System)
* **데이터 영속성 (Data Persistence):** 앱이 종료되거나 기기를 껐다 켜도 저장된 로컬 데이터가 사라지지 않고 유지되는 특성.
