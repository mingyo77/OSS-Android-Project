## 1. Business Purpose

### 1.1 Project Background
현대 외식 산업, 특히 고온의 기름을 사용하는 튀김 전문점의 주방은 극도의 긴장감과 정밀함이 요구되는 공간이다. 돈까스와 같은 메뉴는 원육의 두께, 염지 상태, 튀김기의 온도, 그리고 초 단위로 결정되는 조리 시간에 따라 최종적인 품질이 완전히 달라지는 특성을 지닌다. 그러나 실제 운영 현장에서는 이러한 핵심적인 조리 숙련도가 체계적으로 전수되지 못하고, 선임 작업자의 구두 교육이나 개인의 감각에 의존하는 교육 방식이 주를 이루고 있다.

이러한 방식은 신입 아르바이트생이 투입될 때마다 동일한 교육 리소스를 반복적으로 소모하게 만들며, 바쁜 영업 시간대에는 교육의 질이 급격히 저하되는 고질적인 문제를 야기한다. 특히 주방 특유의 높은 소음과 긴박한 동선 속에서 신입 인력이 두꺼운 종이 매뉴얼을 일일이 확인하며 조리하는 것은 현실적으로 불가능에 가깝다. 이 과정에서 발생하는 조리 시간 미준수, 레스팅 단계 생략 등은 맛의 저하뿐만 아니라 고객 불만족과 식재료 폐기라는 직접적인 경제적 손실로 이어진다.

또한, 주방의 수많은 반복 과업(오픈 준비, 위생 관리, 마감 업무 등)을 단순히 암기나 수동 체크리스트에 의존할 경우 업무의 연속성이 깨지거나 핵심 항목이 누락되는 경우가 빈번하다. 본 프로젝트는 이러한 주방 업무를 '개인의 기억'이 아닌 '시스템의 데이터'로 전환하여, 누구나 첫날부터 숙련공에 가까운 능력을 발휘할 수 있는 표준화된 주방 운영 모델을 제시하고자 한다.

### 1.2 Goals
* **조리 품질의 상향 평준화:** 정밀 타이머와 디지털 레시피를 통해 작업자 개인의 편차와 상관없이 최적의 맛(튀김 온도, 레스팅 시간 준수)을 구현한다.
* **운영 프로세스의 디지털화:** 실시간 체크리스트를 통해 반복되는 업무 누락을 차단하고 업무 연속성을 보장한다.

## 4. Concept of Operation

### 1) 로그인
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 계정 식별을 통해 사용자별 맞춤 환경을 제공하고 보안을 유지한다. |
| **Approach** | 사용자가 입력한 ID와 비밀번호를 등록된 정보와 대조한다. |
| **Dynamics** | 근무 시작을 위해 시스템에 접속을 시도하는 경우. |
| **Goals** | 본인 확인 절차를 거쳐 권한에 맞는 메인 화면으로 진입한다. |

### 2) 오늘의 공지사항
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 사장님이 전달하는 당일의 특이사항이나 재고 현황 등을 누락 없이 파악한다. |
| **Approach** | 메인 화면의 공지사항 탭을 통해 등록된 텍스트 리스트를 출력한다. |
| **Dynamics** | 출근 직후 사장님이 남긴 최신 정보를 확인하고 싶은 경우. |
| **Goals** | 사장님이 작성한 최신 공지 내용을 화면에 명확히 출력한다. |

### 3) 오픈 업무 조회
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 영업 시작 전 반드시 수행해야 할 초기 업무들을 체계적으로 확인한다. |
| **Approach** | 주방 가동 전 준비 단계의 업무 항목들을 리스트 형식으로 나열한다. |
| **Dynamics** | 주방 오픈 전 체크해야 할 업무 리스트를 확인하고 싶은 경우. |
| **Goals** | 수행해야 할 시작 업무들의 목록을 한눈에 시각화하여 표시한다. |

### 4) 메뉴 조회
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 조리 대상 메뉴를 빠르고 정확하게 탐색하여 선택할 수 있게 한다. |
| **Approach** | 매장에서 판매 중인 전체 메뉴 리스트를 화면에 나열한다. |
| **Dynamics** | 특정 메뉴의 조리법을 찾기 위해 목록을 탐색하는 경우. |
| **Goals** | 등록된 전체 메뉴를 사용자가 선택하기 쉬운 리스트로 표시한다. |

### 5) 상세 조리법
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 메뉴별 정밀 조리 데이터(튀김 온도, 레스팅 시간 등)를 숙지한다. |
| **Approach** | 메뉴 선택 시 상세 페이지에서 수치화된 조리 데이터를 출력한다. |
| **Dynamics** | 조리 전 정확한 조리 스펙을 다시 한번 확인해야 하는 경우. |
| **Goals** | 튀김 온도, 시간 등 필수 정보를 상세하게 화면에 출력한다. |

### 6) 조리 타이머
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 설정된 조리 시간을 엄격히 준수하여 조리 품질을 상향 평준화한다. |
| **Approach** | 타이머 버튼 터치 시 설정된 시간부터 카운트다운을 시작한다. |
| **Dynamics** | 돈까스를 기름에 넣는 시점에 정확한 시간 측정이 필요한 경우. |
| **Goals** | 정해진 시간에 맞춰 정밀한 카운트다운 로직을 구동시킨다. |

### 7) 실시간 남은 시간 확인
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 조리 종료 시점을 파악하여 다음 작업을 미리 준비할 수 있게 한다. |
| **Approach** | 시스템 내부의 '초' 데이터를 '분:초' 형식으로 가공하여 표기한다. |
| **Dynamics** | 조리 중 현재 남은 시간을 실시간으로 모니터링하고 싶은 경우. |
| **Goals** | 남은 시간을 00분 00초 포맷으로 가공하여 시각화한다. |

### 8) 업무 완료 상태 기록
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 완료된 업무와 진행 중인 업무를 구분하여 업무 효율성을 높인다. |
| **Approach** | 항목 옆의 체크박스를 클릭하면 해당 데이터의 상태 값을 '완료'로 변경한다. |
| **Dynamics** | 조리 완료 혹은 청소 등 특정 업무를 마친 후 이를 기록하고 싶은 경우. |
| **Goals** | 터치 시 체크 표시와 함께 상태 정보를 완료로 업데이트한다. |

### 9) 체크리스트 자동저장
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 예기치 못한 앱 종료 상황에서도 알바생의 업무 기록이 소실되지 않게 보호한다. |
| **Approach** | 상태가 변경될 때마다 시스템이 즉시 내부 파일 형식으로 데이터를 기록한다. |
| **Dynamics** | 사용자가 체크박스를 클릭하는 등 데이터 변화가 발생한 경우. |
| **Goals** | 변경된 최신 업무 데이터를 로컬 저장소에 자동으로 반영한다. |

### 10) 로그아웃
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 개인 세션을 안전하게 종료하여 타인의 오조작을 방지하고 보안을 확보한다. |
| **Approach** | 로그아웃 버튼 터치 시 현재 계정의 접속을 해제하고 초기 화면으로 전환한다. |
| **Dynamics** | 퇴근 시 혹은 근무 교대 시 시스템 사용을 마무리하는 경우. |
| **Goals** | 세션을 종료하고 메인 로그인 화면으로 사용자를 이동시킨다. |

## 5. Problem Statement
* **로컬 데이터의 영속성 및 동시성 제어:** 알바생이 체크리스트를 업데이트하는 즉시 로컬 파일에 안전하게 기록되어야 한다.
* **복잡한 동선 내 UI/UX 반응성:** 주방 업무 특성상 기름기나 장갑 착용 상태에서도 조작 오류가 없도록 직관적이고 높은 반응성을 확보해야 한다.
* **소음 환경 대응:** 주방 소음으로 인해 알람이 들리지 않는 경우를 대비해 화면 깜빡임 등의 시각적 피드백을 강화해야 한다.
* **사용자 접근성:** 다양한 연령대나 숙련도의 사용자가 별도 교육 없이 기능을 파악할 수 있는 직관적 설계를 지향한다.
